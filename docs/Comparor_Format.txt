Формат программы сравнителя ( следуйте ему при написании своего сравнителя )

Эта программа не должна проверять таймлимит или тому подобное, она только сверяет ответ пользователя
с верным ответом.

Вход ( параметры передаваемые программе, пути передаются абсолютные ):
1) файл верного ответа
2) файл ( неверного ;) ) ответа пользователя

Выход :
Все данные должны выводится в файл cmp.info, который также должна создавать программа сравнителя
1) первая строка файла cmp.info должна содержать единственный символ (без кавычек):
   '+' - тест пройден верно
   '-' - тест пройден неверно
   
Пока в системе Sandra распознаются только эти два символа и соответсвуют полному и нулевому балу за тест.
Однако вы можете вернуть еще какие нибудь значения например типа presentation error ( но символ представляющий 
его должен быть один например 'p'). Но пока за такие значения никакие частичные баллы не присуждаются (то есть это 
всеравно что '-').

Правка 1. Ошибка предстваления (presentation error) теперь оценивается системой в 0.75 балла (макс. 1) и 
обозначается как "*" (точнее символ черепашки)

2) далее, в следующих строках, может следовать (а может и не следовать) любая дополнительная информация 
   про результат прохождения например в чем были отличия файлов или полные файлы. Вся эта информация 
   будет скопирована в лог тестирования.

Упущения :
1) Желательно доделать возможность частичного прохождения тестов типа presentation error в самой Sandr'e и не так
жестко вшивать подсчет балов в систему.
